# 🔐 Implementing a Data Loss Prevention (DLP) Solution at SAGE Books

## 📋 Project Overview

This capstone project presents a comprehensive plan for designing and deploying a **Data Loss Prevention (DLP) solution** for SAGE Books, a multi-location retail bookseller with e-commerce operations. The organization faced significant data breach risks due to insufficient data protection controls and a lack of employee cybersecurity awareness.

The project covers the full lifecycle of a DLP implementation — from root cause analysis and stakeholder mapping through phased deployment, risk management, and workforce training.

---

## 🎯 Problem Statement

SAGE Books lacked robust mechanisms to protect sensitive customer and financial data, leaving the organization exposed to:

- Unauthorized data access and exfiltration
- Regulatory penalties (GDPR, PCI DSS non-compliance)
- Insider threats stemming from untrained staff
- Reputational and financial damage from a potential breach

**Root causes identified:**
1. Insufficient cybersecurity awareness training across the workforce
2. Inadequate security expertise within the information security team
3. Ad hoc, inconsistently delivered cybersecurity awareness programs

## 🏗️ Methodology & Frameworks

The implementation strategy was guided by the following industry standards:

| Framework | Application |
|---|---|
| **NIST SP 800-53** | Security control selection and implementation |
| **NIST Cybersecurity Framework (CSF)** | Risk-based Identify → Protect → Detect → Respond → Recover model |
| **ISO/IEC 27001** | Information security management system structure |
| **PCI DSS** | Cardholder data protection for payment card transactions |
| **GDPR** | Data privacy compliance for EU customer data |

Project management followed an **Agile methodology**, enabling iterative delivery and continuous stakeholder feedback.

---

## 🗂️ Project Phases

### Phase 1 — Planning & Assessment
- Current-state assessment of data security practices
- Stakeholder identification and project team formation
- Risk assessment: identified breach scenarios and vulnerabilities
- Regulatory requirements mapping (GDPR, PCI DSS)

### Phase 2 — Solution Selection & Design
- Vendor evaluation based on features, scalability, compatibility, and cost
- DLP architecture design and configuration planning
- Policy and ruleset definition for data classification, monitoring, and enforcement

### Phase 3 — Implementation & Deployment
- Pilot testing in a controlled environment
- Staged rollout across endpoints, network, and cloud environments
- User training and awareness program delivery
- Integration with existing security infrastructure

### Phase 4 — Monitoring & Optimization
- Continuous log, alert, and report monitoring
- Periodic policy and configuration reviews
- Compliance audits and KPI tracking
- Stakeholder feedback loops for ongoing improvement

## ⚠️ Key Implementation Risks

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Resource Constraints | Moderate–High | High | Early budget planning; phased deployment to manage scope |
| Technical Compatibility Issues | High | High | Thorough compatibility assessments; engage IT architects pre-deployment |

## 🎓 Training Plan

| Audience | Format | Duration |
|---|---|---|
| All Employees | General awareness session | 1–2 hours |
| IT Team | Hands-on technical training (multi-session) | Several days |
| Managers & Supervisors | Leadership workshop | Half-day |

**Training content areas:** DLP fundamentals, policy & compliance requirements, technical configuration, data handling best practices, role-based scenarios.
